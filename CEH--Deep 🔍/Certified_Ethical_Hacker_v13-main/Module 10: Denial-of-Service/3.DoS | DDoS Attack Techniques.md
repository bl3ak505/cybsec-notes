# 💥 DoS/DDoS Attack Techniques
Attackers implement various techniques to launch denial-of-service (DoS)/distributed denial-of-service (DDoS) attacks on target computers or networks. This section discusses the basic categories of DoS/DDoS attack vectors, various attack techniques, and various DoS/DDoS attack tools used to take over a single or multiple network system to exhaust their computing resources or render them unavailable to their intended users.

---
## Basic Categories of DoS/DDoS Attack Vectors 
DDoS attacks mainly aim to diminish the network bandwidth by exhausting network, application, or service resources, thereby restricting legitimate users from accessing system or network resources. In general, DoS/DDoS attack vectors are categorized as follows: 

### Volumetric Attacks 
**Volumetric Attacks** are a type of **DDoS attack** that flood a target’s network or service with massive traffic, using up all available bandwidth and blocking real users from accessing it. The attack strength is measured in **bits per second (bps)**.

These attacks often abuse protocols like **NTP**, **DNS**, and **SSDP**, which don’t have built-in limits to prevent overloading. Since one machine alone can't generate enough traffic, attackers use multiple compromised systems to send huge volumes of data at once. This overwhelms network devices like routers and switches, causing service disruption and traffic congestion across the network.

There are two types of bandwidth depletion attacks: 
- In a flood attack, zombies send large volumes of traffic to the victim’s systems to exhaust the bandwidth of these systems.
- In an amplification attack, the attacker or zombies transfer messages to a broadcast IP address. This method amplifies malicious traffic that consumes the bandwidth of the victim’s systems.

Attackers use botnets and perform DDoS attacks by flooding the network. The entire bandwidth is used up by attackers, and no bandwidth remains for legitimate use. The following are examples for volumetric attack techniques:
```
- User Datagram Protocol (UDP) flood attack			- NTP amplification attack
- Internet Control Message Protocol (ICMP) flood attack		- Spoofed IP packet flood attack
- Ping of Death (PoD) attack					- Malformed IP packet flood attack
- Smurf attack							- Zero-day attack
- Pulse wave attack
```
### Protocol Attacks
**Protocol Attacks** are a type of DDoS attack that don’t just consume bandwidth—they target and exhaust system resources like **connection state tables** on devices such as **firewalls, load balancers, and servers**. These tables keep track of active connections, and when they're full, no new connections can be made.

Attackers send a massive number of fake requests to fill up these tables, causing real users to be blocked. The attack is measured in **packets per second (pps)** or **connections per second (cps)** and can overwhelm even high-capacity systems.

The following are examples for protocol attack techniques:
```
- Synchronize (SYN) flood attack		- TCP SACK panic attack	
- Fragmentation attack				- RST attack
- Spoofed session flood attack			- TCP state exhaustion attack
- Acknowledgement (ACK) flood attack		- TCP connection flood attack
- SYN-ACK flood attack				- ACK and PUSH ACK flood attack
```

### Application Layer Attacks
**Application Layer Attacks** target the software or application itself rather than the network. Attackers exploit bugs or weak points in the code to overload the application with fake requests, making it unavailable to real users. These attacks don’t need much traffic, so even one or two machines can cause serious damage.

Attackers might flood websites with fake user activity, send too many invalid login attempts, or use malicious SQL queries to crash databases. One common method is **buffer overflow**, where too much data is sent to an app, causing it to crash or run harmful code.

These attacks are hard to detect because the traffic looks normal. They’re measured in **requests per second (rps)** and can cause service outages, financial loss, and reputational damage. However, if detected, they’re easier to trace back to the source compared to other DDoS types.

The following are examples for application layer attack techniques:
```
- Hypertext Transfer Protocol (HTTP) flood attack
- Slowloris attack
- UDP application layer flood attack
- DDoS extortion attack
```

## DoS/DDoS Attack Techniques 
Next, the following DoS/DDoS attack techniques will be discussed: 

### UDP Flood Attack
**UDP Flood Attack** is a type of denial-of-service (DoS) attack where the attacker sends a huge number of fake UDP packets to random ports on a target server using different source IPs. The server tries to respond to each request, but finds no real service running on those ports, so it replies with error messages. This process wastes system resources and internet bandwidth, eventually making the network slow or completely unavailable for real users.

---
### ICMP Flood Attack 
**ICMP Flood Attack** is a type of Denial-of-Service (DoS) attack where attackers send a massive number of ICMP echo request packets (pings) to a target system. These requests force the system to reply, overloading its network and making it unable to respond to real users.

This attack can be done directly or through reflection networks. To prevent it, network devices can be configured with a **threshold limit** (usually 1000 packets per second). If this limit is crossed, the system blocks further ICMP requests from that source for the rest of the current second and the next one, helping to protect against the flood.

![image](https://github.com/user-attachments/assets/1f9cddac-01f5-4f5a-bb62-0b01aee801ed)

---
### Ping of Death Attack 
**Ping of Death (PoD)** is a type of attack where the attacker sends an oversized or malformed ping packet—larger than the allowed **65,535 bytes**—to a target system. When the target tries to reassemble this large packet, it may crash, freeze, or become unstable. The attacker only needs the target's **IP address**, and their identity can be easily hidden (spoofed). This makes PoD a simple but disruptive denial-of-service attack.

![image](https://github.com/user-attachments/assets/eae9c53e-cfe4-4d94-a12f-bee1ef926d57)

---
### Smurf Attack
In a Smurf attack, the attacker spoofs the source IP address with the victim’s IP address and sends a large number of ICMP ECHO request packets to an IP broadcast network. This causes all the hosts on the broadcast network to respond to the received ICMP ECHO requests. These responses are sent to the victim’s machine because the IP address was spoofed by the attacker, causing significant traffic to the victim’s machine and ultimately making it crash.

![image](https://github.com/user-attachments/assets/b4bcf826-a9b8-40b2-be58-9da3db7d00de)

---
### Pulse Wave DDoS Attack 
**Pulse Wave DDoS Attack** is a powerful form of DDoS attack where attackers send massive bursts of traffic (called pulses) at regular intervals instead of continuous flow. These high-speed pulses, often **300 Gbps or more**, hit the target every **10 minutes** and can last for **hours or even days**. Each pulse overloads the target’s network, making recovery extremely hard or sometimes impossible. This method is designed to keep services offline repeatedly and cause maximum disruption.

![image](https://github.com/user-attachments/assets/a45a7537-a6bc-4d0f-9122-64554db45c9e)

---
### Zero-Day DDoS Attack 
**Zero-Day DDoS Attacks** are powerful attacks that target unknown vulnerabilities in a system—ones that have no patch or defense yet. Attackers use these flaws to overwhelm and block access to a victim’s network or services while stealing sensitive data. Since the vulnerability is not yet identified or fixed, the victim can’t stop the attack until it’s too late. These attacks can cause serious damage, and there’s currently no universal method to fully protect against them.

---
### NTP Amplification Attack
**NTP Amplification Attack** is a type of **DDoS attack** that misuses the **Network Time Protocol (NTP)**, which is meant for syncing time on network devices. In this attack, the attacker sends fake requests to an NTP server using the **victim’s IP address**. These requests use the **monlist** command, which triggers large responses from the server.

Since the responses are much larger than the original requests, the victim’s system gets **flooded with massive traffic**, overloading the network. This leads to **service disruption, network slowdown**, and drains system resources like **bandwidth and memory**. The attack is powerful because a small effort from the attacker can cause a large-scale impact.

![image](https://github.com/user-attachments/assets/bc7b44f7-e6ef-4c68-ac93-0617c3f07fa6)

**Steps to retrieve monlist from an NTP server:**
Run the following command to retrieve the monlist. 
```bash
nmap -sU -pU:123 -Pn -n --script=ntp-monlist <target>
```
After executing the above command, Nmap provides a list of public and private clients that have communicated with the NTP server.

**Countermeasures for NTP amplification attack:**

- Secure and harden NTP server configurations to prevent monlist exploitation
- Limit flow control in the NTP server
- Frequently monitor network for abnormal behavior
- Implement zero-trust network
- Use firewalls to filter NTP server requests

### SYN Flood Attack
**SYN Flood Attack** is a type of DoS (Denial of Service) attack where an attacker sends a massive number of fake **SYN requests** to a server. Normally, a TCP connection uses a three-step handshake: **SYN → SYN/ACK → ACK**. In a SYN flood, the attacker sends only the SYN request with a fake IP and never completes the handshake.

The server, waiting for the final ACK, keeps the connection open in its memory. When too many of these incomplete connections pile up, the server’s queue gets full, and it can’t accept new real connections—causing disruption. This slows down or crashes the server. The attack is hard to trace because it often uses spoofed IPs and can also be done using variations like **SYN-ACK** and **ACK/PUSH ACK floods**.

![image](https://github.com/user-attachments/assets/be2d972e-2ee4-4068-a994-920160575aca)

#### SYN-ACK Flood Attack
This type of attack is similar to the SYN flood attack, except that in this type of flood attack, the attacker exploits the second stage of a three-way handshake by sending a large number of SYN-ACK packets to the target machine to exhaust its resources.

#### ACK and PUSH ACK Flood Attack
During an active TCP session, ACK and PUSH ACK are the flags used to transfer information to and from the server and client machines till the session ends. In an ACK and PUSH ACK flood attack, attackers send a large amount of spoofed ACK and PUSH ACK packets to the target machine, making it non-functional.

#### Countermeasures for SYN Flood Attacks
**Countermeasures for SYN Flood Attacks** involve filtering traffic and fine-tuning system settings to reduce damage. Admins can use tools like **SYN cookies** and **SynAttackProtect** to block fake connection attempts. They can also lower the time a server waits for a connection in the **"SYN RECEIVED"** state and limit or disable packet retransmissions to prevent bandwidth overload. These steps help protect servers from being overwhelmed while still allowing real users to connect.

### Fragmentation Attack 
**Fragmentation Attack** is a method where attackers send many large, broken-up (fragmented) packets to a target system to overload it. These packets, often over 1500 bytes, pass through network defenses like firewalls and IDS/IPS without being checked, because the system expects to reassemble them later.

The attacker randomizes the packet content, making it harder for the system to put them back together. This consumes a lot of memory and processing power, slowing down the system or even causing it to crash. It's a stealthy way to reduce performance without needing high-speed traffic.

![image](https://github.com/user-attachments/assets/f38812cc-883b-4eb8-816b-abab86aac7d7)

---
### Spoofed Session Flood Attack 
In this type of attack, attackers create fake or spoofed TCP sessions by carrying multiple SYN, ACK, and RST or FIN packets. Attackers employ this attack to bypass firewalls and perform DDoS attacks against target networks, exhausting their network resources.

The following are examples for spoofed session flood attacks: 

- **Multiple SYN-ACK Spoofed Session Flood Attack** - In this type of flood attack, attackers create a fake session with multiple SYN and multiple ACK packets, along with one or more RST or FIN packets.
- **Multiple ACK Spoofed Session Flood Attack** - **Multiple ACK Spoofed Session Flood Attack** is a type of DDoS attack where the attacker skips the usual SYN packets used to start a connection and sends only fake **ACK**, **RST**, or **FIN** packets to flood the target. Since most firewalls are designed to detect attacks by filtering **SYN** packets, they often miss this type of attack, making it harder to detect and block.

### HTTP GET/POST Attack
**HTTP GET/POST attacks** are advanced **Layer 7 (application layer)** DoS attacks that target web servers by misusing normal-looking HTTP requests.

In an **HTTP GET attack**, the attacker sends a request with delayed headers and never completes it. The server keeps the connection open, waiting, which wastes resources and blocks real users.

In an **HTTP POST attack**, the attacker sends full headers but only part of the message body. Again, the server waits for the rest, using up memory and resources.

These attacks don’t use fake packets or high bandwidth—they just exhaust the server by forcing it to stay busy with incomplete requests, making it slow or unavailable to legitimate users.

![image](https://github.com/user-attachments/assets/3d51f38e-7e5f-480e-ab8a-180169ef68e0)

In addition to the aforementioned HTTP GET/POST attack, attackers can employ the following HTTP flood attacks to exhaust the target network’s bandwidth: 

* **Single-Session HTTP Flood Attack**
  The attacker sends many requests using just **one open HTTP session**. This takes advantage of HTTP 1.1 to overload the server without opening multiple connections.

* **Single-Request HTTP Flood Attack**
  The attacker hides **multiple HTTP requests in one packet**, making it hard to detect. It helps them stay hidden while launching DDoS attacks.

* **Recursive HTTP GET Flood Attack**
  The attacker acts like a real user by requesting **pages or images one after another**, but it’s actually a flood attack disguised to bypass firewalls.

* **Random Recursive GET Flood Attack**
  A more advanced version of recursive GET, where the attacker uses **random page numbers** (like in blogs or forums) to look even more real, sending endless GET requests to drain server resources.

### Slowloris Attack
**Slowloris** is a type of DDoS attack that targets the application layer (Layer 7) by sending **incomplete HTTP requests** to a web server. The server keeps these connections open, waiting for the rest of the request. As the attacker keeps sending more partial requests without finishing them, the server eventually runs out of available connections. This makes the server unable to handle real user requests, effectively taking it down using what looks like normal traffic.

### Multi-Vector Attack 
In multi-vector DDoS attacks, the attacker uses combinations of volumetric, protocol, and application layer attacks to take down the target system or service. The attacker quickly changes from one form of DDoS attack (e.g., SYN packets) to another (layer 7). These attacks are either launched through one vector at a time or through multiple vectors in parallel to confuse a company’s IT department, making them spend all their resources and maliciously diverting their focus.

### Peer-to-Peer Attack
**Peer-to-Peer (P2P) Attack** is a type of DDoS attack where attackers abuse bugs in P2P file-sharing networks, like those using the **Direct Connect (DC++) protocol**, to flood a target website. Instead of using a botnet, the attacker tricks many users in a P2P network into disconnecting and connecting to the victim’s site, causing heavy traffic that slows down or crashes the site.

This attack is easy to spot using known traffic patterns and can be reduced by blocking P2P traffic on common web ports like **port 80** to protect websites from being overwhelmed.

![image](https://github.com/user-attachments/assets/30a50f4f-6995-4837-9771-cd563d04af7b)

### Permanent Denial-of-Service Attack
**Permanent DoS (PDoS)**, also called **phlashing**, is a powerful attack that damages hardware permanently. Unlike normal DoS attacks that just slow down or crash a system, PDoS makes the device completely unusable, often requiring full replacement.

Attackers exploit security flaws in devices like routers or printers to install **fake hardware updates** through emails, chats, or pop-ups. These updates contain **corrupted or harmful firmware**, and when the victim installs them, the attacker gains control and "bricks" the device—making it stop working entirely. PDoS is fast, destructive, and doesn’t need many resources to be effective.

![image](https://github.com/user-attachments/assets/4b83486e-218b-46b3-ade1-b5975b7cb295)

### TCP SACK Panic Attack
**TCP SACK Panic Attack** is a remote attack that targets Linux systems using TCP Selective Acknowledgment (SACK). It works by sending specially crafted SACK packets with a very small **maximum segment size (MSS)** to overload the system’s **socket buffer**, which stores unacknowledged data.

Linux can only handle up to **17 segments** in this buffer. When attackers force the system to handle more than that, it causes an **integer overflow** in the buffer, leading to a **kernel panic** and crashing the system. This results in a **Denial of Service (DoS)**. Since this flaw exists in the Linux kernel, the attack can also affect **containers and virtual machines**.

![image](https://github.com/user-attachments/assets/c5ed8c75-ba13-45a4-9a6b-62d6f9249cf5)

#### Countermeasures
- Implement vulnerability patching
- Implement a firewall rule to block requesting packets with the lowest MSS

### Distributed Reflection Denial-of-Service (DRDoS) Attack 
**Distributed Reflection Denial-of-Service (DRDoS) Attack** is a smart and powerful type of DDoS attack where the attacker hides behind other systems to flood the target. The attacker first controls some machines (called **zombies**) to send fake connection requests using the target’s IP to innocent systems (called **reflectors**). These reflectors believe the target asked for a connection, so they send back a flood of **SYN/ACK** responses to the target.

The target, which never made the request, is overwhelmed by this traffic and becomes slow or completely unresponsive. Since the traffic comes from trusted systems (reflectors), it’s hard to trace back to the real attacker. This method creates a much larger and harder-to-stop attack than a typical DDoS.

![image](https://github.com/user-attachments/assets/94c9f6f7-7744-403a-8a49-383e3dd6a83d)

**Countermeasures**
- Turn off the Character Generator Protocol (CHARGEN) service to stop this attack method
- Download the latest updates and patches for servers

### DDoS Extortion/Ransom DDoS (RDDoS) Attack 
**DDoS Extortion** or **Ransom DDoS (RDDoS)** is an attack where hackers threaten organizations with a powerful DDoS attack unless a ransom is paid. They often send a warning email or launch a small sample attack to prove they’re serious. The ransom note usually demands payment in cryptocurrency and includes a deadline, along with threats of exposing vulnerabilities or data. In many cases, attackers fake their power by claiming they have advanced tools, pressuring the victim to pay out of fear of major business disruption.

![image](https://github.com/user-attachments/assets/75276d48-beae-405e-b7b5-a157b2bc718c)

**Countermeasures**
- Implement effective DDoS defense tools
- Immediately report to the law enforcement agencies and security teams after receiving a ransom note
- Frequently evaluate assets for risk tolerance
- Implement mitigation strategies such as BGP/ DNS swing and always-on protection service
