# 🤖 Botnets
The term “bot” is a contraction of “robot” and refers to software applications that run automated tasks over the Internet. Attackers use bots to infect a large number of computers that form a network, or “botnet,” allowing them to launch DDoS attacks, generate spam, spread viruses, and commit other types of crime. 

This section deals with organized cyber-crime syndicates, organizational charts, botnets, and botnet propagation techniques; botnet ecosystems; scanning methods for finding vulnerable machines; and the propagation of malicious code.

---
## Organized Cyber Crime: Organizational Chart 
**Organized Cyber Crime** involves cybercriminals working together in structured groups, much like a company. These groups follow a clear hierarchy and share profits from illegal activities. They rent out tools like **botnets**, create **malware**, hack **bank accounts**, and launch **DDoS attacks** for money.

For example, a gang might launch a DDoS attack on a bank to distract its security team while they steal money using stolen login details. Some of these groups are also involved in **cyber warfare** and **hacktivism**, making them a serious threat to national security.

The structure looks like this:
- **Boss** – the head who runs operations like a business but doesn’t do the attacks directly.
- **Underboss** – sets up command-and-control servers and crimeware tools.
- **Campaign Managers** – lead attack operations and manage teams.
- **Resellers** – sell the stolen data for profit.

Each member is paid based on their role and task in the cybercrime chain.

## Botnets
**Botnets** are networks of computers infected with bots—automated programs that can be used for both good and bad purposes. While some bots collect data for harmless tasks like web crawling, others are used by hackers to launch attacks or steal data.

A **botnet** (a contraction of “roBOT NETwork”) is a group of infected systems controlled remotely by an attacker. Even a small botnet can generate powerful attacks like **DDoS**. Botnets are central to cybercrime, often used to spread **malware** such as **Trojans, viruses, worms, and keyloggers**.

Cybercriminals buy and sell botnet services to perform illegal tasks like scanning systems for vulnerabilities, stealing data, or attacking servers—all while staying hidden. Examples of bots include **Cardinal**, **Sopel**, and **Eggdrop**, often used in chat or IRC environments.

Attackers can use botnets to perform the following: 
1. **DDoS Attacks** - Botnets flood a server with massive traffic to crash websites or disrupt online services.
2. **Spamming** - Attackers use bots to send out bulk spam emails using stolen email addresses.
3. **Traffic Sniffing** - Bots monitor network traffic to steal sensitive data like passwords or credit card numbers.
4. **Keylogging** - Records everything typed on the keyboard to steal login info, especially for financial sites.
5. **Spreading Malware** - Botnets install and spread new malware to infect more systems and grow the botnet.
6. **Click Fraud** - Bots auto-click ads to generate fake revenue or inflate ad performance statistics.
7. **Credential Stuffing** - Bots use stolen usernames and passwords to try logging into various websites automatically.
8. **Crypto Mining** - Bots secretly use a victim’s computer power to mine cryptocurrency for the attacker.
9. **Online Poll/Game Manipulation** - Bots vote or perform automated actions to unfairly influence poll results or game outcomes.
10. **Mass Identity Theft** - Bots send phishing emails pretending to be from trusted companies (e.g., eBay) to steal identities.

These botnet activities are powerful and dangerous because they can be automated and scaled across thousands or millions of infected machines.

## Scanning Methods for Finding Vulnerable Machines
Discussed below are scanning methods used by an attacker to find vulnerable machines in a network: 

### Random Scanning 
**Random Scanning** is a method where an infected device (controlled by an attacker) randomly scans IP addresses in a network to find vulnerable systems. When it finds one, it infects it with the same malware. These newly infected systems then continue the process, spreading the malware further.

In the beginning, this spreads quickly, but over time, it slows down as fewer new IP addresses are left to target. This method also causes a lot of network traffic because many infected machines scan the same IPs repeatedly.

### Hit-list Scanning
**Hit-list Scanning** is a fast and aggressive method attackers use to infect multiple systems. First, the attacker creates a list of vulnerable machines (the "hit list"). When one machine is compromised, the attacker installs malicious code on it and splits the hit list. The attacker scans one half, while the newly infected machine scans the other half. This cycle repeats, with each new infected system helping to spread the attack. As a result, the number of infected machines grows rapidly, allowing malware to spread across the entire list in a short time.

### Topological Scanning
**Topological Scanning** is a method where an infected machine looks through its own files (like browser history or stored URLs) to find new targets to attack. It uses this information to locate other systems that might be vulnerable and then tries to infect them. This method is smart and accurate because it focuses on real connections the user has, and it works similarly to hit-list scanning.

### Local Subnet Scanning
In this technique, an infected machine searches for new vulnerable machines in its local network, behind a firewall, by using the information hidden in the local addresses. Attackers use this technique in combination with other scanning mechanisms.

### Permutation Scanning
**Permutation Scanning** is a smart scanning method used by attackers to find new targets without scanning the same systems repeatedly. All infected machines use a shared, shuffled list of IP addresses made with a 32-bit block cipher and a secret key.

When a system gets infected, it starts scanning the list from a specific point—either right after itself or from a random spot—depending on how it was infected. If it finds another already-infected machine, it jumps to a new random point. If it keeps hitting infected systems without finding new ones, it stops scanning and creates a new key to start fresh. This makes the scan faster and harder to detect.

Permutation scanning has the following advantages:
- The reinfection of a target is avoided.
- New targets are scanned at random, thereby ensuring a high scanning speed.

## How Does Malicious Code Propagate? 
Discussed below are three techniques used by an attacker to propagate malicious code and build attack networks: 

### Central Source Propagation 
**Central Source Propagation** is a method where attackers store their attack tools on a central server. When they find a vulnerable system, they command the server to send a copy of the toolkit to that system. The toolkit is then automatically installed using scripts.

The newly infected system then starts looking for more vulnerable machines, spreading the attack further in a cycle. This technique commonly uses **HTTP**, **FTP**, and **RPC** protocols to transfer and install the tools.

![image](https://github.com/user-attachments/assets/039cce49-20b3-4696-87ef-44140935fca6)

### Back-chaining Propagation
**Back-chaining Propagation** is a method where an attacker installs attack tools on their own system and then sends a copy to other vulnerable systems they find. The compromised system connects back to the attacker’s machine to receive these tools, often using **TFTP** (Trivial File Transfer Protocol). This technique helps spread the malware or toolkit from one system to another, using simple port listeners or attacker-controlled web servers.

![image](https://github.com/user-attachments/assets/7ef03a55-b905-40ca-9c44-8c86fa24fce5)

### Autonomous Propagation
**Autonomous Propagation** is a self-spreading attack method where the infected system automatically transfers the attack tools to any new vulnerable system it finds, right at the moment it breaks in. Unlike other methods that rely on external sources to send the tools, here the attacker’s system does everything on its own, making the spread faster and harder to stop.
