# 🌐 Gain Remote Access
After vulnerabilities are found in the scanning phase, an attacker can remotely access an IoT device and turn it into a hidden backdoor. Because these devices often sit inside networks and may bypass endpoint protections, attackers can use them to move laterally, maintain command-and-control access, and launch further attacks within the organization without touching protected endpoints. This makes securing IoT devices critical to prevent unauthorized network access and wider compromise.

---
## Gaining Remote Access using Telnet 
Attackers scan networks to find open Telnet ports on IoT devices like routers, cameras, or industrial controllers. When Telnet is open, they can probe the device to learn its model and software, then decide how to proceed. If the service has no authentication, the attacker gains immediate access; if it requires credentials, they try default logins (e.g., `root/root`) or brute-force passwords. Tools such as Shodan and Censys make finding exposed Telnet services easy. Once inside, attackers can read or alter device data, move laterally, or install malware, so disabling Telnet and using strong authentication are essential defenses.
