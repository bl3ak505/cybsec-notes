# 🔍 Information Gathering 
Information gathering is the first step in targeting an OT network, where attackers use footprinting and reconnaissance to map the environment. They enumerate devices, find geolocations, discover open ports and services, and collect default or weak credentials to identify attack paths. Tools like Shodan and Nmap are commonly used to locate connected OT devices and gather these details before moving to later stages of an attack.

---
## Identifying ICS/SCADA Systems using Shodan 
🔗Source: [https://www.shodan.io] <br>
Shodan is an internet search engine that indexes devices exposed online, and attackers can use it to find ICS/SCADA systems connected to the Internet. By searching for device banners and protocols, Shodan can reveal control systems used in water treatment, power grids, HVAC, and industrial control units, including exposed RTUs and PLCs. Finding these devices helps attackers map critical infrastructure and look for weak or unprotected systems.

- **Identifying SCADA systems using port numbers** <br>
  SCADA/ICS devices use well-known protocol ports (for example Modbus on 502, Fieldbus 1089–1091, DNP/DNP3 on 19999/20000, Ethernet/IP on 2222, PROFINET on 34962–34964, and EtherCAT on 34980). Scanning the Internet for these ports lets attackers find exposed control systems, which can be vulnerable if left unprotected. Identifying open protocol ports helps defenders quickly locate and secure internet-facing SCADA devices, while attackers can misuse this information to target critical industrial systems. <br>
  Search for Modbus-enabled ICS/SCADA systems: `port:502 (Retrieves all the ICS/SCADA systems with Modbus port 502 enabled)`

- **Discovering SCADA systems using PLC name** <br>
  Attackers can find SCADA systems by searching for identifiable information exposed in device banners—like PLC names, version numbers, or manufacturer strings—using internet scanners such as Shodan. These banners often reveal details (for example, Schneider Electric or Modbus-related identifiers) that let an attacker locate and target industrial devices connected to the internet. <br>
  Search for SCADA systems using PLC name: For example, the search string `“Schneider Electric”` displays all systems that deploy Schneider Electric products.

- **Searching SCADA systems based on geolocation** <br>
  Search for SCADA systems using geolocation:
  SCADA Country:`"US"` (displays all SCADA systems present in the US)

## Gathering Default Passwords using CIRT.net 
🔗Source: [https://www.cirt.net] <br>
CIRT.net’s default password database is an online database that stores default passwords for various devices, including those used in critical infrastructures. Attackers can use this database to obtain various default passwords for a wide range of devices such as routers, switches, industrial control systems (ICS), and more.

## Scanning ICS/SCADA Systems using Nmap 
Attackers use scanning tools such as Nmap to identify open ports and running services on systems connected to OT networks. Discussed below are various Nmap commands used by attackers to enumerate open ports and services of ICS/SCADA systems: 
- **Identifying Open Ports and Services** <br>
  ```bash
  nmap -Pn -sT --scan-delay 1s --max-parallelism 1 -p 80, 102, 443, 502, 530, 593, 789, 1089-1091, 1911, 1962, 2222, 2404, 4000, 4840, 4843, 4911, 9600, 19999, 20000, 20547, 34962-34964, 34980, 44818, 46823, 46824, 55000-55003 <Target IP>
  ```
  Attackers use the above Nmap command to perform initial reconnaissance to identify active ICS/SCADA protocols. The port numbers listed in the command are well-known port numbers used for ICS/SCADA protocols

- **Identifying HMI Systems** <br>
  ```bash
  nmap -Pn -sT -p 46824 <Target IP>
  ```
  Some vendors provide HMI interfaces that operate on ports other than ICS/SCADA ports. For example, consider the HMI software Sielco Sistemi Winlog, which uses TCP port 46824.

- **Scanning Siemens SIMATIC S7 PLCs** <br>
  ```bash
  nmap -Pn -sT -p 102 –script=s7-info <Target IP>
  ```
  Attackers use the above command to detect PLC devices with open port 102. Siemens SIMATIC S7 PLC devices use port 102 for S7 Communication, used for exchanging information between PLC devices and SCADA systems.

- **Scanning Modbus Devices** <br>
	```bash
 	nmap -Pn -sT -p 502 --script modbus-discover <Target IP>
 	nmap -sT -Pn -p 502 --script modbus-discover --script-args='modbus-discover.aggressive=true' <Target IP>
	```
 	Attackers use the above command to identify Modbus-enabled devices along with their Slave IDs.

- **Scanning BACnet Devices** <br>
	```bash
 	nmap -Pn -sU -p 47808 --script bacnet-info <Target IP>
 	```
 	Attackers enumerate BACnet devices used for interconnecting and controlling building and automation systems, HVAC systems, etc. The above command helps attackers to retrieve information such as the name of the vendor, device name, serial number, and firmware version. It also helps attackers to detect BACnet Broadcast Management Devices (BBMDs) using NSE script BACnet-discover-enumerate.nse.

- **Scanning Ethernet/IP Devices** <br>
	```bash
 	nmap -Pn -sU -p 44818 --script enip-info <Target IP>
	```
 	Ethernet/IP is a popular protocol implemented by many industrial networks. Ethernet/IP uses Ethernet as a transport layer protocol, and CIP is used to provide services for industrial applications. This protocol operates on UDP port number 44818. Using the above command, attackers can gather information such as the name of the vendor, product code and name, device name, IP address, etc.

- **Scanning Niagara Fox Devices** <br>
	```bash
 	nmap -Pn -sT -p 1911,4911 --script fox-info <Target IP>
 	```
 	Niagara Fox is a protocol used for device-to-device within building management systems (BMSs). This protocol operates on TCP ports 1911 and 4911. The above command allows attackers to gather information such as application name, Java version, host OS, time zone, local IP address, and software versions.

- **Scanning ProConOS Devices** <br>
	```bash
 	nmap -Pn -sT -p 20547 --script proconos-info <Target IP>
 	```
 	ProConOS is a high-performance runtime engine for PLC devices designed to control embedded and PC-based control applications. Attackers can use the above command to enumerate information such as PLC type, project name, project source code name, and ladder logic runtime information.

- **Scanning Omron PLC Devices** <br>
	```bash
 	nmap -Pn -sT -p 9600 --script omron-info <Target IP>
 	nmap -Pn -sU -p 9600 --script omron-info <Target IP>
 	```
 	Factory Interface Network Service (FINS) is an Omron protocol used by PLC programs to communicate program data and perform other services with a remote PLC device. FINS uses TCP or UDP port 9600 to provide services.

- **Scanning PCWorx Devices** <br>
	```bash
 	nmap -Pn -sT -p 1962 --script pcworx-info <Target IP>
 	```
 	PCWorx are PC-based automated solutions for industrial networks. These systems process unauthenticated messages from remote systems. Attackers use the above command to gather information such as PLC type, model number, and firmware version.

## Sniffing using NetworkMiner 
🔗Source: [https://www.netresec.com] <br>
NetworkMiner is a passive network-forensics tool used to sniff and analyze network traffic without generating additional packets. It extracts details like open ports, hostnames, operating systems, and sessions, and can parse PCAP files to reassemble transferred files or certificates. Attackers can use it to examine past captures from an ICS network to learn about devices and communications, while defenders use the same capabilities for incident investigation and threat hunting.

## Analyzing Modbus/TCP Traffic using Wireshark 
🔗Source: [https://www.wireshark.org] <br>
Wireshark can capture Modbus/TCP traffic on industrial networks, letting an attacker read every packet because Modbus/TCP has no built-in encryption. By inspecting these clear-text messages, an attacker can learn device addresses, function codes, and data values, then craft and send malicious Modbus requests or replay captured packets to disrupt or control equipment. This lack of transport security makes Modbus/TCP easy to monitor and manipulate.

## Discovering ICS/SCADA Network Protocols using Malcolm 
🔗Source: [https://cisagov.github.io] <br>
Malcolm is a network traffic analysis tool that helps identify protocols used in ICS/SCADA environments. It provides visibility through the OpenSearch dashboard for easy protocol overview and Arkime for investigating network sessions and potential security incidents. Malcolm also supports secure communication from the interface and remote log forwarders, making it useful for monitoring and analyzing industrial network traffic.
