# ♻️ Sniffing Technique: ARP Poisoning 
This section discusses the ARP poisoning technique generally used by attackers to perform sniffing on a target network. Using this method, an attacker can steal sensitive information, prevent network and web access, and perform DoS and MITM attacks using sniffing.

---
## What Is Address Resolution Protocol (ARP)?
**Address Resolution Protocol (ARP)** is a network protocol used to find the **MAC address** of a device based on its **IP address**. When a computer wants to send data, it needs the MAC address of the destination device. If it doesn’t know it, it sends an ARP request asking, “Who has this IP?” The device with that IP replies with its MAC address. The system then stores this in the **ARP table**, which keeps track of IP-to-MAC mappings to avoid repeating the process. Both host machines and switches use ARP to communicate on a network.

**The process of obtaining the MAC address using ARP is as follows:**

* The source device creates an ARP request with its own MAC/IP and the target IP, then sends it to the switch.
* The switch checks if the source MAC is in its CAM table; if not, it adds it with the port info.
* The switch broadcasts the ARP request to all devices in the network.
* Each device checks if the target IP in the request matches its own IP.
* The device with the matching IP sends back an ARP reply with its MAC address.
* The switch updates its CAM table with the reply info and forwards it to the original sender.
* The sender updates its ARP table with the MAC address of the destination, and communication starts.

## ARP Spoofing Attack
**ARP Spoofing** is a type of cyberattack where an attacker tricks a device into sending data to the wrong machine by sending fake ARP (Address Resolution Protocol) messages. ARP maps IP addresses to MAC addresses on a network, but it doesn’t verify who sends the response.

Attackers send **forged ARP replies** with fake IP and MAC addresses, and the victim’s device accepts them as real. This updates the ARP table with wrong information, allowing the attacker to **intercept, monitor, or manipulate** the victim’s data. This is called **ARP cache poisoning** and can lead to further attacks like **Man-in-the-Middle (MITM)**, **Denial of Service (DoS)**, or **session hijacking**, all without the victim knowing.

## Threats of ARP Poisoning 

* **Packet Sniffing**: Attacker captures and reads private data moving through the network.
* **Session Hijacking**: Attacker steals active session info (like login sessions) to take unauthorized control.
* **VoIP Call Tapping**: Attacker listens and records voice calls by intercepting VoIP traffic.
* **Manipulating Data**: Attacker captures and changes data or blocks it from reaching the destination.
* **Man-in-the-Middle Attack**: Attacker sits between two systems and secretly relays or alters communication.
* **Data Interception**: Attacker gathers IPs, MAC addresses, and VLAN info from the network.
* **Connection Hijacking**: Attacker takes control of an ongoing network connection.
* **Connection Resetting**: Misleading network data can cause a system to break or drop connections.
* **Stealing Passwords**: Attacker tricks users into sending sensitive data like usernames and passwords.
* **DoS Attack**: Attacker floods the target with too much traffic, making it crash or stop working.

## ARP Spoofing/Poisoning Tools

### arpspoof 
🔗Source: [https://linux.die.net]

**arpspoof** is a tool used to redirect network traffic by sending fake ARP replies on a local network. It tricks a target device into thinking the attacker’s system is the gateway, so all traffic meant for the real gateway is sent to the attacker instead.

For example, using the command:
`arpspoof -i [Interface] -t [Target Host]`
the attacker can place themselves between the victim and the gateway.

This allows the attacker to **sniff, monitor, or even alter** the data being sent. By sending ARP replies in both directions, the attacker stays in the middle of the communication, making this a powerful method for **Man-in-the-Middle (MITM)** attacks.

## How to Defend Against ARP Poisoning
**To defend against ARP poisoning**, networks use a feature called **Dynamic ARP Inspection (DAI)**. DAI checks if ARP packets are valid by comparing them to trusted data stored in a **DHCP snooping table**, which keeps records of IP and MAC addresses from DHCP exchanges. This prevents fake ARP messages that could lead to **man-in-the-middle (MITM)** attacks.

Before enabling DAI, **DHCP snooping must be turned on**. If not, devices in the VLAN may lose connection, causing a self-inflicted DoS. If a device uses a **static IP**, DHCP snooping won’t work, so **static IP-to-MAC mapping** must be used to prevent ARP spoofing.

Additionally, custom **ARP monitoring scripts** can be used to regularly check for mismatched MAC/IP pairs. If a mismatch is found, the system drops the packet, helping protect key devices like servers and gateways from ARP poisoning.

### Configuring DHCP Snooping and Dynamic ARP Inspection on Cisco Switches
**DHCP Snooping** is a security feature on Cisco switches that monitors DHCP traffic and builds a table of trusted IP-to-port mappings. It helps prevent attackers from sending fake DHCP responses.

Before enabling **Dynamic ARP Inspection (DAI)**, DHCP snooping must be turned on. DAI uses the snooping table to block malicious ARP messages and protect against ARP spoofing. Together, these features secure the network at Layer 2 by verifying DHCP and ARP traffic.

To configure DHCP snooping on a Cisco switch, ensure DHCP snooping is enabled both globally and per access VLAN. To enable DHCP snooping, execute the following commands:
```bash
# 1. Enable DHCP snooping globally and on VLAN 10
Switch# config terminal
Switch(config)# ip dhcp snooping
Switch(config)# ip dhcp snooping vlan 10

# 2. Trust the uplink port so real DHCP replies are allowed (e.g., Fa0/5)
Switch(config)# interface FastEthernet0/5
Switch(config-if)# ip dhcp snooping trust
Switch(config-if)# exit

# 3. Exit config mode and verify DHCP snooping status
Switch(config)# end
Switch# show ip dhcp snooping
# → Confirms snooping is on for VLAN 10

# 4. View the snooping binding table (MAC↔IP bindings)
Switch# show ip dhcp snooping binding
# → Lists trusted clients’ MAC/IP and lease info

# 5. Enable Dynamic ARP Inspection (DAI) on VLAN 10
Switch# config terminal
Switch(config)# ip arp inspection vlan 10
Switch(config)# end

# 6. Check ARP inspection status and drop counters
Switch# show ip arp inspection
# → Shows validation settings and counts of forwarded vs. dropped ARP packets
```
