# 🌐 Sniffing Technique: DNS Poisoning  
This section describes DNS poisoning techniques to sniff the DNS traffic of a target network. Using this technique, an attacker can obtain the ID of the DNS request by sniffing and can send a malicious reply to the sender before the actual DNS server responds.

---
## DNS Poisoning Techniques
**DNS Poisoning**, also known as **DNS Spoofing**, is a technique where an attacker tricks a DNS server into redirecting users to a fake, malicious website instead of the real one. DNS normally converts domain names like **[www.eccouncil.org](http://www.eccouncil.org)** into IP addresses like **208.66.172.56** using DNS tables. In poisoning, the attacker manipulates these tables to insert a fake IP address.

When a victim tries to visit a website, their system is unknowingly sent to the attacker’s server, which may look like the real site. The attacker can then steal data, install malware, or monitor the victim’s activity—all without the victim noticing.

DNS poisoning is possible using the following techniques:
- Intranet DNS Spoofing
- Internet DNS Spoofing
- Proxy Server DNS Poisoning
- DNS Cache Poisoning

---
## Intranet DNS Spoofing 
**Intranet DNS Spoofing** is an attack where the hacker tricks a system on a local network into connecting to a fake website. Using **ARP poisoning**, the attacker must be inside the same LAN and able to sniff network traffic. Once they capture a **DNS request**, they quickly send a fake reply with a malicious IP address before the real DNS server responds. This redirects the user to a harmful site without their knowledge.

The diagram describes how an attacker performs an intranet DNS spoofing.

The attacker tricks the router using `arpspoof` and `dnsspoof` tools to redirect DNS requests from the victim (John) to their own fake website. When John asks for the IP of a real site (like `www.xsecurity.com`), the attacker gives a fake IP pointing to a malicious site. John unknowingly visits the fake site and enters sensitive info like passwords. The attacker captures this data, then silently forwards John to the real site—so John doesn't suspect anything.

This is a classic **DNS spoofing and phishing attack** using ARP poisoning.

## Internet DNS Spoofing 
**Internet DNS Spoofing**, also called **remote DNS poisoning**, is a type of **Man-in-the-Middle (MITM)** attack where the attacker tricks a victim’s computer into using a fake DNS server. This is usually done using a **Trojan** when the victim connects to the internet.

The attacker sets up a **rogue DNS server** with a fixed IP address and changes the victim’s DNS settings to point to it. As a result, when the victim tries to visit a real website, they are silently redirected to a fake or malicious site controlled by the attacker. This lets the attacker **steal sensitive information** like login credentials or personal data.

## Proxy Server DNS Poisoning
**Proxy Server DNS Poisoning** is a technique where the attacker sets up a fake proxy server with a malicious DNS. Using a **Trojan**, the attacker changes the victim’s proxy settings so that all internet traffic goes through this fake proxy. The proxy then acts like the victim’s DNS, redirecting them to **fake websites** that look real. Here, the attacker can **steal sensitive information** like passwords or personal data before silently forwarding the victim to the actual website.

## DNS Cache Poisoning 
**DNS Cache Poisoning** is a cyberattack where fake DNS records are inserted into a DNS resolver’s cache. This tricks the system into redirecting users to a malicious site instead of the real one.

When a user types a domain name, the DNS resolver usually checks its cache for speed. If an attacker has already inserted a fake IP address into the cache, the resolver sends users to the attacker’s fake site. This happens because the resolver fails to verify if the DNS response is from a trusted source. As a result, users are unknowingly redirected to harmful websites, putting their data at risk.

## SAD DNS Attack
**SAD DNS Attack** is a modern type of **DNS cache poisoning** where attackers insert fake DNS records into a server’s cache. This tricks users into visiting **malicious websites** instead of real ones.

Attackers exploit **side-channel flaws** and weaknesses in DNS software like **dnsmasq**, **unbound**, and **BIND**, even on updated systems. The goal is to redirect traffic to attacker-controlled servers, where users may unknowingly download harmful files or reveal sensitive information.

## How to Defend Against DNS Spoofing
Major DNS implementations have reported attacks using DNS spoofing, and this vulnerability still affects a large number of organizations. This is because of a lack of information when performing DNS queries, which allows attackers to spoof DNS responses.

Countermeasures that help prevent DNS spoofing attacks are as follows but not lemeted:
- Implement Domain Name System Security Extensions (DNSSEC).
- Use a Secure Socket Layer (SSL) for securing the traffic.
- Resolve all DNS queries to a local DNS server.
- Block DNS requests to external servers.
- Configure a firewall to restrict external DNS lookup.
- Implement an intrusion detection system (IDS) and deploy it correctly.
- Configure the DNS resolver to use a new random source port for each outgoing query.
- Restrict the DNS recusing service, full or partial, to authorized users.
- Use DNS nonexistent domain (NXDOMAIN) rate limiting.
