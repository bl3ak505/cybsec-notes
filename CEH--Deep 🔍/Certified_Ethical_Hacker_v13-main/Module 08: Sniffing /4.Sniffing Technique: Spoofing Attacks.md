# 🕵️‍♂️ Sniffing Technique: Spoofing Attacks 
Besides ARP spoofing, an attacker can also use MAC spoofing, IRDP spoofing, VLAN hopping, and STP attacks to sniff the traffic of a target network. This section describes spoofing techniques that help attackers to steal sensitive information. This section also explains how to defend against MAC spoofing, VLAN hopping, and STP attacks.

---
## MAC Spoofing/Duplicating 
**MAC Spoofing/Duplicating** is a technique where an attacker copies the MAC address of a real user on a network to pretend to be them. The attacker first listens to the network to find active MAC addresses, then changes their device's MAC to match one of those.

If successful, the network sends data meant for the real user to the attacker instead. This allows the attacker to access the network and steal the user’s identity and data.

## MAC Spoofing Technique: Windows 
There are two methods for MAC spoofing in Windows 11 OS: 

### Method 1: If the network interface card supports clone MAC address, then follow these steps:
1. Click on **Start**, search for **Control Panel** and open it, then navigate to **Network and Internet** → **Networking and Sharing Center**.
2. Click on **Ethernet** and then click on **Properties** in the **Ethernet Status** window.
3. In the **Ethernet Properties** window, click on the **Configure** button and then on the **Advanced** tab.
4. Under the **“Property”** section, browse for **Network Address** and click on it.
5. On the right-hand side, under **“Value,”** type in the new MAC address you would like to assign and click **OK**.
  > Note: Enter the MAC address number without “:” in between.
7. Type **“ipconfig/all”** or **“net config rdr”** in the command prompt to verify the changes.
8. If the changes are visible, then **reboot** the system, or else try method 2 (change MAC address in the registry).

![image](https://github.com/user-attachments/assets/455ea535-985c-4245-83b0-211bf6be8b25)

### Method 2: Steps to change the MAC address in the registry: 
1. Press `Win + R` to open **Run**, and type `regedit` to start the **registry editor**.
2. Go to `"HKEY_LOCAL_MACHINE\SYSTEM\CurrentControlSet\Control\Class\{4d36e972-e325-11ce-bfc1-08002be10318}` and double-click on it to expand the tree.
3. Four-digit sub keys representing network adapters will be found (starting with 0000, 0001, 0002, etc.).
4. Search for the proper **“DriverDesc”** key to find the desired interface.
5. Right-click on the appropriate sub key and add the new string value **“NetworkAddress”** (data type “REG_SZ”) to contain the new MAC address.
6. Right-click on the **“NetworkAddress”** string value on the right side and select **Modify**...
7. Now, in the **“Edit String”** dialog box, enter the new MAC address in the **“Value data”** field and click **“OK.”**
8. **Disable** and then **re-enable** the network interface that was changed, or **reboot** the system.

## IRDP Spoofing
**IRDP Spoofing** is a network attack where an attacker sends fake router advertisement messages to trick devices into using the attacker’s system as the default route. Since **IRDP (ICMP Router Discovery Protocol)** doesn’t require authentication, the victim's system accepts these spoofed messages—especially if the attacker sets a high preference and lifetime for the route.

This only works if the attacker is on the same network as the victim. Once the fake route is accepted, all traffic goes through the attacker’s machine. This allows the attacker to **sniff data**, perform **Man-in-the-Middle (MITM)** attacks, or cause **Denial of Service (DoS)** by sending wrong routes.

To stay safe, **disable IRDP** on systems if possible, especially when it’s not needed.

## VLAN Hopping
**VLAN Hopping** is a network attack where an attacker tricks the system to jump from one VLAN to another, gaining access to restricted network traffic. This usually happens due to **poor configuration** or **weak VLAN setup**.

The attacker bypasses network isolation and can then **steal sensitive data**, **inject malware**, or **spread viruses** across the network. VLAN hopping breaks the security meant to separate different parts of a network, making it a serious threat if not properly secured.

VLAN hopping attacks can be performed via two primary methods, as given below:

### Switch Spoofing
**Switch Spoofing** is a technique where an attacker connects a fake (rogue) switch to the network and tricks a real switch into forming a **trunk link**—a connection that carries traffic from multiple VLANs. Once the trunk is established, the attacker can capture and analyze traffic from different VLANs.

This attack works only if the real switch is set to **negotiate trunking**, using modes like **dynamic auto**, **dynamic desirable**, or **trunk**. If these settings are enabled, the attacker can easily intercept and view sensitive network data.

![image](https://github.com/user-attachments/assets/492d3cc2-cf10-4e4f-ac76-bfdb9eb73881)

### Double Tagging
**Double Tagging** is a VLAN hopping attack where an attacker sneaks traffic into another VLAN by adding two VLAN tags to an Ethernet frame. The outer tag matches the attacker’s native VLAN, so the first switch removes it and forwards the frame. The inner tag, which belongs to the target VLAN, is left intact and gets delivered to the victim VLAN.

This trick lets the attacker bypass VLAN isolation and send data to other VLANs. It only works if the switch ports use **native VLANs**, making it a misconfiguration-based attack.

## STP Attack
**STP Attack** targets the **Spanning Tree Protocol**, which is used to prevent loops in a LAN by choosing the best path for network traffic. In this protocol, one switch is selected as the **root bridge**, and all traffic routes are based on it. The root is chosen using **Bridge Protocol Data Units (BPDUs)**, which contain a **Bridge ID (BID)** made from a priority number and MAC address.

In an STP attack, an attacker adds a **rogue switch** with a very **low priority value**, making it become the new **root bridge**. Once this happens, the attacker’s switch becomes the main traffic path, allowing them to **capture and sniff all the network traffic**. This compromises the network’s security and performance.

## How to Defend Against MAC Spoofing
**Defending Against MAC Spoofing** involves identifying weak points in the network and applying the right countermeasures. After testing for MAC spoofing, the goal is to secure the network by controlling which devices can connect and how.

The most effective way is placing servers behind routers, as routers use IP addresses, not MAC addresses. Also, enabling **port security** on switches allows only specific MAC addresses to connect and can take action if violations occur.

Other strong defenses include **DHCP snooping**, **Dynamic ARP Inspection (DAI)**, and **IP Source Guard**, which help detect and block spoofed MAC and IP addresses. Tools like **802.1X**, **AAA servers**, and **Network Access Control (NAC)** add strong access control.

Using **WPA3 encryption**, retrieving MAC addresses directly from hardware, and performing **regular audits** also help strengthen protection. Finally, **rate limiting** and **traffic analysis** can reduce the impact of spoofing and flooding attacks.

## How to Defend Against VLAN Hopping

### Defend Against Switch Spoofing
Defend Against Switch Spoofing Perform the following steps to configure a switch to prevent switch spoofing attacks: 

- Explicitly configure the ports as access ports, and ensure that all access ports are configured not to negotiate trunks: 
  ```bash
  switchport mode access
  switchport mode nonegotiate
  ```
- Ensure that all trunk ports are configured not to negotiate trunks:
  ```bash
  switchport mode trunk
  switchport mode nonegotiate
  ```

### Defend Against Double Tagging 
Perform the following steps to configure a switch to prevent double tagging attacks:
- Ensure that you specify the default VLAN, which is used if the interface stops trunking:
  ```bash
  switchport access vlan 2
  ```
- Ensure that the native VLANs on all trunk ports are changed to an unused VLAN ID:
  ```bash
  switchport trunk native vlan 999
  ```
- Ensure that the native VLANs on all trunk ports are explicitly tagged:
  ```bash
  vlan dot1q tag native
  ```

Some additional measures to defend against double tagging attacks are as follows:
- **Use Private VLANs:** Configure private VLANs to isolate ports from each other on the same VLAN.
- **Regularly Audit and Monitor VLAN Configurations:** Perform regular audits of VLAN and switch configurations to ensure compliance with security policies.

## How to Defend Against STP Attacks
Implement the following countermeasures to defend against STP attacks on switches:

### BPDU Guard
**BPDU Guard** is a network security feature used to protect **PortFast-enabled ports**, which are meant for end devices—not other switches. If a **BPDU (Bridge Protocol Data Unit)** is received on such a port, it means a switch might be wrongly connected. To prevent network loops, **BPDU Guard automatically shuts down the port** by placing it in **errdisable mode**, stopping all traffic on that port. This helps keep the network stable and secure.

Use the following commands to enable BPDU guard on a switch interface: 
```bash
configure terminal
interface gigabitethernet slot/port
spanning-tree portfast bpduguard
```

### Root Guard
**Root Guard** is a security feature in STP (Spanning Tree Protocol) that protects the main switch, called the **root bridge**, from being replaced. It ensures the current root stays in control of the network.

If a connected switch tries to take over by sending a **superior BPDU**, the port with Root Guard goes into a special **loop-inconsistent state** (not completely shut down), stopping that switch from affecting the STP topology. The port stays inactive only for that switch until the problem is fixed, helping keep the network stable and secure.

Use the following commands to enable the root guard feature on a switch interface:
```bash
configure terminal
interface gigabitethernet
slot/port spanning-tree guard root
```

### Loop Guard
Loop guard improves the stability of the network by preventing it against the bridging loops. It is generally used to protect against a malfunctioned switch. 

Use the following commands to enable the loop guard feature on a switch interface: 
```bash
configure terminal
interface gigabitethernet
slot/port spanning-tree guard loop
```

### UDLD (Unidirectional Link Detection)
UDLD enables devices to detect the existence of unidirectional links and further disable the affected interfaces in the network. These unidirectional links in the network can cause STP topology loops. 

Use the following command to enable UDLD on a switch interface: 
```bash
configure terminal
interface gigabitethernet
slot/port udld { enable | disable | aggressive }
```

Some additional measures to defend against STP Attacks are as follows:
- **Deploy PortFast:** Apply PortFast to all access ports to reduce the time spent in listening and learning STP states. However, ensure that BPDU Guard is enabled to mitigate risks.
- **Regularly update and patch network devices:** Update firmware and software on network devices regularly to protect against known vulnerabilities.
- **Restrict network access:** Limit physical access to network ports and devices to prevent unauthorized STP configuration changes.
- **Network segmentation:** Limit the scope of STP attacks by dividing larger broadcast domains into smaller, manageable segments.
