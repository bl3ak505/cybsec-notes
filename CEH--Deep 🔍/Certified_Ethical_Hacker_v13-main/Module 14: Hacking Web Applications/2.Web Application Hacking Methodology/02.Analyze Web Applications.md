# 🕵️‍♂️ Analyze Web Applications 
After attacking the server, attackers often focus on the web application to find weaknesses. They study how the application works, understand its features and technologies, and look for areas that may be vulnerable. Even a single flaw can be enough for them to launch a successful attack. By analyzing the application, they reduce the attack surface and increase their chances of exploiting it effectively.

Analyzing the target website will typically provide the following information: 
- **Software used and its version:** An attacker can easily find the software and version in use on an off-the-shelf software-based website.

- **Operating system used:** Usually, the operating system in use can also be determined.

- **Sub-directories and parameters:** Searches can reveal the sub-directories and parameters by making a note of the URLs while browsing the target website.

- **Filename, path, database field name, or query:** The attacker will often carefully analyze anything after a query that looks like a filename, path, database field name, or query to check whether it offers opportunities for SQL injection.

- **Scripting platform:** With the help of script filename extensions such as .php, .asp, or .jsp, one can easily determine the scripting platform that the target website is using.

- **Technologies Used:** By inspecting the URLs of the target website, one can easily determine the technologies (.NET, J2EE, PHP, etc.) used to build that website.

- **Contact details and CMS details:** The contact pages usually offer details such as names, phone numbers, email addresses, and locations of admin or support personnel. An attacker can use these details to perform a social engineering attack. CMS software allows URL rewriting to disguise the script filename extensions if the attacker is willing to devote additional effort toward determining the scripting platform.

- **SSL certificate details:** The attacker can access information about the target website's security certificate, including the issuer, expiration date, and encryption strength.

- **Cookies:** By analyzing the cookies set by the target website, the attacker can gain insights into the user sessions, preferences, and tracking mechanisms employed.

- **Error messages:** Examining the error messages returned by the web server may disclose information about the underlying technologies, file paths, or server configurations.

- **Publicly accessible files:** Attackers can access publicly accessible files in the target website which may contain sensitive information, such as configuration files, backup files, or log files.


The following are various techniques for analyzing web applications: 

1. **Identify Entry Points for User Input** → First, find where users can enter data in the app (like forms, URLs, or APIs). These are potential targets for attacks, so attackers analyze the HTTP requests to spot them.

2. **Identify Server-Side Technologies** → Find out which server-side languages or frameworks the app uses (like PHP, ASP.NET, JSP). Attackers use techniques like HTTP fingerprinting to detect them and plan suitable attacks.

3. **Identify Server-Side Functionality** → Understand what the server does in the background when a user interacts with the app. This includes running scripts to generate content. Attackers analyze this behavior to find weak points.

4. **Identify Files and Directories** → Look for hidden or sensitive files and folders that might be exposed due to misconfiguration. Tools like Gobuster help attackers discover these and gather more information.

5. **Identify Web Application Vulnerabilities** → Check the app for security flaws caused by poor coding practices. These flaws can be exploited for various types of attacks.

6. **Map the Attack Surface** → Finally, list all the possible points of attack based on the findings. This helps attackers focus on the most vulnerable areas of the app.

## Techniques to Gather Information for Web Application Analysis 

### Extracting Website Links 
Extracting website links is a key part of website footprinting, where attackers collect all internal and external URLs from a target site. This helps them discover related resources like JavaScript, CSS files, and third-party services. By analyzing these links, attackers can uncover technologies in use, find hidden pages, and identify potential vulnerabilities. Tools like Octoparse and Link Extractor are often used to automate this process and gather more detailed information for further exploitation.

- **Octoparse** [https://www.octoparse.com] <br>
  Octoparse offers automatic data extraction, as it quickly scrapes web data without coding and turns web pages into structured data. As shown in the screenshot, attackers use Octoparse to capture information from webpages, such as text, links, image URLs, or html code.

### Gathering the Wordlist from the Target Website
Attackers collect words from a target website to create custom wordlists that help in brute-force attacks. They often first gather email addresses linked to the organization using search engines or social media, then use tools like CeWL to extract words from the website. These words are then used to guess passwords or access credentials linked to the collected email addresses.

To run the CeWL tool, issue the following commands: 
```bash
cewl --help	# → This command displays various options that a user can use to obtain a list of words from the target website.

cewl https://www.certifiedhacker.com	# → This command returns a list of unique words present in the target URL.
```

### Extracting Metadata of Public Documents
Extracting metadata from public documents is a method attackers use to gather hidden details from files like PDFs or Word documents available on a target’s website. These files often contain information such as usernames, email addresses, document authors, and timestamps. Attackers analyze this data to plan brute-force attacks or social engineering campaigns, increasing the chances of compromising the organization’s systems.

#### Metadata Extraction Tools 
Metadata extraction tools such as ExifTool, Web Data Extractor Pro, and Metagoofil automatically extract critical information that includes the usernames of clients, OSes (exploits are OS-specific), email addresses (possibly for social engineering), list of software used (version and type), list of servers, document creation/modification date, and website authors. 

- **ExifTool** [https://exiftool.org] <br>
	ExifTool is a cross-platform Perl library and command-line application for reading, writing, and editing metadata in files. It supports a large number of metadata formats, including EXIF, HTML, GPS, IPTC, XMP, and JFIF, and allows copying metadata between files of different formats. The tool also helps in extracting thumbnail images, preview images, and large JPEG images from RAW files.

### Monitoring Web Pages for Updates and Changes
Attackers monitor websites to detect any updates or changes, such as modifications in login pages, software versions, or protected content. By tracking these changes, they gather useful information to uncover potential vulnerabilities. This helps them plan and carry out targeted attacks on the web application.

#### Web Updates Monitoring Tools 
Web updates monitoring tools such as WebSite-Watcher, Visualping, and Follow That Page are capable of detecting any changes or updates on a particular website, and they can send notifications or alerts to interested users through email or SMS. 

- **WebSite-Watcher** [https://www.aignes.com] <br>
	WebSite-Watcher helps to track websites for updates and automatic changes. When an update or change occurs, WebSite-Watcher automatically detects and saves the last two versions onto your disk. ### Searching for Contact Information, Email Addresses, and Telephone Numbers from Company Website
Attackers can search the target company’s website to gather crucial information about the company. Generally, organizations use websites to inform the public about what they do, what type of services or products they provide, how to contact them, etc. Attackers can exploit this information to launch further attacks on the target company. <br> 
For example, attackers can search for the following information on the company’s website:
- Company contact names, phone numbers, and email addresses
- Company locations and branches
- Partner Information
- News
- Links to other sites
- Product, project, or service data

### Searching for Web Pages Posting Patterns and Revision Numbers
Attackers can search for copyright notices and document revision numbers on a target organization's website to gather detailed information. Copyright notices often reveal the author's name, creation year, and ownership details, while revision numbers and document IDs help track changes in published materials. This information can be used to analyze the organization’s internal structure and potentially find weak points for further attacks.

### Monitoring Website Traffic of the Target Company 
Attackers can monitor a target company’s website traffic using tools such as Web-Stat, Rank Tracker, and TeamViewer to collect valuable information. These tools help to collect information about the target’s customer base, which help attackers to disguise themselves as customers and launch social engineering attacks on the target.

The information collected includes:
- **Total visitors:** Tools such as **Clicky** [https://clicky.com] find the total number of visitors browsing the target website.

- **Page views:** Tools such as **Opentracker** [https://www.opentracker.net] monitor the total number of pages viewed by the users along with the timestamps and the status of the user on a particular web page (whether the webpage is still active or closed).

- **Bounce rate:** Tools such as **Google Analytics** [https://analytics.google.com] measure the bounce rate of the target company’s website.

- **Live visitors map:** Tools such as **Web-Stat** [https://www.web-stat.com] track the geographical location of the users visiting the company’s website.
  
- **Site ranking:** Tools such as **Rank Tracker** [https://www.ranktracker.com] track a company’s rank on the web.

- **Audience geography:** Tools such as **Rank Tracker** track a company’s customer locations on the globe.

- **Track Visitors and conversation:** Tools such as **Dashly** [https://www.dashly.io] track visitors and show conversation rates with the company’s website.

## Website Mirroring
Website mirroring is the process of copying an entire website to a local system using tools like HTTrack or Cyotek WebCopy. Attackers use this method to analyze the site offline, study its structure, and look for vulnerabilities without interacting with the live server. This helps them plan and launch targeted attacks on the original website.

### Website Mirroring Tool: HTTrack Web Site Copier 
🔗Source: [https://www.httrack.com] <br>
HTTrack is a website mirroring tool that lets users download an entire website to a local system. Attackers often use it to create a full offline copy of a target organization's site. This allows them to study the site’s structure, browse it locally, and look for potential vulnerabilities or exploits without alerting the organization.

## Identify Entry Points for User Input 
Attackers identify entry points for user input to understand how a web application receives and processes data. They analyze elements like URLs, HTTP headers, query strings, POST data, and cookies to locate all possible input fields. By examining these areas, they look for weaknesses such as poor input validation or encoding issues, which can lead to injection attacks. This helps them find ways to bypass security and gain unauthorized access to the application.

## Identify Server-Side Technologies 
- Perform detailed server fingerprinting and analyze the HTTP headers and HTML source code to identify server-side technologies
- Examine URLs for file extensions, directories, and other identification information
- Examine the error page messages
- Examine session tokens: JSESSIONID – Java, ASPSESSIONID – IIS server, ASP.NET_SessionId – ASP.NET, PHPSESSID – PHP
- Use tools such as httprint and WhatWeb to identify server-side technologies

## Identify Files and Directories
Attackers use various techniques and tools to enumerate applications, hidden directories, and files of the web application hosted on web servers that are exposed on the Internet. They use tools such as Gobuster and URL Fuzzer and the Nmap NSE script http-enum to identify files and directories of the target web application.

- **Gobuster** [https://github.com/OJ/gobuster] <br>
	Gobuster is a Go-programming-based directory scanner that allows attackers to perform fast-paced enumeration of hidden files and directories of a target web application. It is a command-oriented tool used to brute-force URIs in websites, DNS subdomains, names of virtual hosts on the target server, etc. <br>
Run the following command to retrieve file and directory names and their status codes:
```bash
gobuster -u <target URL> -w common.txt			# -> Scans the target URL using wordlist 'common.txt' to find hidden directories/files.

gobuster -u <target URL> -w common.txt -s 200	# -> Same as above, but only shows results with HTTP status code 200 (OK).

# Additional Flags
-q   # Quiet mode – hides status codes and banners for cleaner output.
-n   # No status – does not show the status codes in the output.
-o   # Output – saves the scan results to a file. Example: -o result.txt
```

- **Nmap** [https://nmap.org] <br>
Attackers use the Nmap NSE script http-enum to enumerate applications, directories and files of web servers that are exposed on the Internet. Thus, they can identify critical security vulnerabilities in the target web application. <br>
Run the following Nmap command to gather information about the exposed files and directories of the target web server:
```bash
nmap -sV --script=http-enum <target domain or IP address>
```

## Identify Web Application Vulnerabilities 
Attackers identify web application vulnerabilities by scanning the application to find weak points that can be exploited to gain admin access or steal sensitive data. They use automated tools to detect flaws in the code, components, or technologies used. These scans help attackers map out vulnerabilities, assess their severity, and plan their attacks accordingly.
Attackers can use tools such as Vega, WPScan Vulnerability Database, Arachni, and Uniscan to identify vulnerabilities in the target web applications. 

- **Vega** [https://www.subgraph.com] <br>
  	Vega is a free, open-source tool used to scan web applications for security issues. It helps detect vulnerabilities like SQL injection, XSS, shell injection, and more. Built in Java with a graphical interface, Vega works on Linux, macOS, and Windows. It can also check TLS/SSL settings to find ways to improve server security.
