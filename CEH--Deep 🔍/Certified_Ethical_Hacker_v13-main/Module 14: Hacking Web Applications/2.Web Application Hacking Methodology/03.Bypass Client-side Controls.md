# 🤡 Bypass Client-side Controls
Bypassing client-side controls involves exploiting weak validation done on the user's browser before data is sent to the server. Developers often use techniques like hidden form fields or scripts to limit user input, but if these checks aren't reinforced on the server side, attackers can easily manipulate the data. This can lead to security issues such as data tampering or unauthorized access.

Common web application attacks used to bypass client-side controls: 
```
▪ Cross-site scripting (XSS)			▪ Local storage/session storage manipulation
▪ Cross-site request forgery (CSRF)		▪ JavaScript injection
▪ Parameter tampering				▪ Cross-origin resource sharing (CORS) misconfiguration
▪ HTTP parameter pollution			▪ Session fixation 
▪ Input field manipulation			▪ Attack browser extensions		
▪ Hidden field manipulation			▪ Client-side input validation bypass
```

---
## Attack Hidden Form Fields
Attackers target hidden form fields in web applications to manipulate data like product prices or discount rates. These fields are meant to be hidden from users but still exist in the webpage’s source code, making them easy to access and change using browser tools. If sensitive information is stored in these fields without proper server-side validation, attackers can alter values like prices before submitting the form, leading to unauthorized benefits or fraud.

Follow the process described below to attack hidden form fields: 
- Identify vulnerable web applications
- Save the source code for the HTML page 
- Locate the hidden field 
- Tamper with the price values by editing the price field’s value 
- Save the file and reload the source into a browser 
- Click the Buy button

The request will be transmitted to the server with the modified price. You can also use proxy tools such as Burp Suite to trap the request that submits the form and modify the price field to any value. In addition, you can attempt to enter negative price values to trick the retail application into refunding the amount through credit card transactions.

## Attack Browser Extensions
The data from a web application that uses browser extension components can be captured by two methods: 
- **Intercepting Traffic from Browser Extensions** <br>
	Attempt to intercept and modify the request and response of the component and the server, respectively. You can use tools such as Burp Suite to capture the data. This method has certain limitations such as data obfuscation or encryption, and secure data serialization.

- **Decompiling Browser Extensions** <br>
	Decompiling browser extensions allows you to view their source code and understand how they work, even if the code is obfuscated. This helps in modifying the data sent to the server, bypassing client-side validations. Tools like Burp Suite can intercept and alter these requests, just like with regular HTML forms, making it possible to test or exploit the behavior of browser extensions during communication with the server.

## Attack Google Chrome Browser Extensions
To attack Google Chrome, attackers trick users into downloading malware or visiting a malicious website. Once the browser is infected, it can force users to install harmful extensions. These extensions may abuse Chrome’s Sync feature to steal sensitive data like saved passwords, search history, bookmarks, autofill details, and other browser settings by appearing as legitimate tools.

- Cookies, pixel tags, application cache, browser web storage, and server log files

## Perform Source Code Review
Source code review involves analyzing the actual code of a web application to find security weaknesses. By reviewing the code, it's possible to understand how different components and frameworks work, and to spot vulnerabilities like weak input validation, insecure data handling, or exposed internal functions. This helps in identifying flaws that may not be visible during regular testing.

Perform source code review to identify the following functionalities of a target component:
- Client-side input validation or other security-related logics and events
- Obfuscation or encryption techniques that are applied to the client data before it is transmitted to the server
- Modifiable components with hidden client-side functionalities
- References to server-side functionalities
