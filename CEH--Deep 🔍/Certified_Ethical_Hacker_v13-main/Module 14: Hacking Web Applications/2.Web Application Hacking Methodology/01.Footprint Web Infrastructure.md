# 🔎 Footprint Web Infrastructure
Web infrastructure footprinting is the process of collecting detailed information about how a web application is built and connected to other systems. This is usually the first step attackers take to understand the technologies used, how everything is linked, and where weaknesses might exist. By studying the web infrastructure, attackers can find potential entry points to exploit and gain unauthorized access to the application.

Footprinting the web infrastructure allows an attacker to engage in the following tasks: 

1. **Server discovery** → Attackers try to find out which physical servers are hosting the web application using tools like Whois, DNS lookups, and port scanning.

2. **Service discovery** → They identify which services (like HTTP, FTP, SSH) are running on the server to find potential entry points for attacks.

3. **Server identification** → Attackers perform banner grabbing to learn the web server type and version, and gather details like server location, internal IPs, and protocol support.

4. **WAF and proxy detection** → They check if the web application uses a firewall or proxy, helping them understand what security is in place.

5. **Hidden content discovery** → Attackers search for hidden pages or functionalities not directly visible to users, which may reveal sensitive information.

6. **Load balancers detection** → They try to detect load balancers and uncover real IPs of backend servers that may be exposed online.

7. **Web app technologies detection** → They identify the technologies used in building the web app (e.g., PHP, ASP.NET, WordPress) to find specific vulnerabilities.

8. **WebSocket enumeration** → Attackers detect active WebSocket servers that might be used for real-time communication and could be vulnerable.

## Server Discovery
To footprint a web infrastructure, first, you need to discover active Internet servers. Three techniques, namely Whois lookup, DNS interrogation, and port scanning, help in discovering the active servers and their associated information. 

### Whois Lookup 
Whois lookup tools allow you to gather information about a domain with the help of DNS and Whois queries. They provide information about the IP address of the web server and DNS names. These tools produce results in the form of an HTML report. 

Use the following tools to perform Whois lookup: 
- **Netcraft** [https://www.netcraft.com]
- **Whois Lookup** [https://whois.domaintools.com]
- **Batch IP Converter** [http://www.sabsoft.com]
- **Whois Domain Lookup** [https://www.whois.com]
- **Whois** [https://webwhois.verisign.com]
- **Whois Lookup** [https://mxtoolbox.com]

### DNS Interrogation
DNS interrogation is a technique used to gather information about an organization’s network by querying its Domain Name System (DNS), which links IP addresses to hostnames. If the DNS is not properly secured, attackers can exploit it to discover server locations, types, and other sensitive details. This information can then be used to plan and launch targeted cyberattacks.

Use the following tools to perform DNS interrogation:
- **DNSRecon** [https://www.kali.org/tools/dnsrecon/]
- **DNS Records** [https://www.nslookup.io]
- **Domain Dossier** [https://centralops.net]
- **DNSdumpster.com** [https://dnsdumpster.com]

### Server Discovery: Banner Grabbing
Banner grabbing is a technique used by attackers to gather information about a target system by sending a request and analyzing the response. The server replies with a banner that often reveals details like the software name, version, or operating system. Hackers use this information to find known vulnerabilities and choose the right exploits to attack the server or its applications.

How an attacker can use telnet to establish a connection and gain banner information of a target is demonstrated below: 

1. **Open Command Prompt (CMD):**
   The attacker opens the command prompt on their system.

2. **Run Telnet Command:**
   They use this command to connect to the target's web server:

   ```
   telnet moviescope.com 80
   ```

   * `moviescope.com` can also be an IP address.
   * `80` is the port used by HTTP.

3. **Connection Established:**
   After running the command, the screen goes blank (no response shown). This is normal.

4. **Press Esc Key:**
   Pressing the **Esc** key sends a signal to the server, which then responds with a **banner message**.

5. **Banner Info Revealed:**
   This message may include:

   * Web server type (e.g., Apache, Nginx)
   * Server version
   * Operating system
   * Other details

### Grabbing Banners from SSL Services 
Banner grabbing over HTTP can be done easily using tools like Telnet or Netcat, but these tools don't work for SSL-encrypted connections like HTTPS. Since SSL encrypts the data, attackers use tools like OpenSSL to connect securely and extract server banners or details from SSL services. This helps them identify server information even over encrypted connections.

Attackers perform the following steps to grab banners over an SSL connection: 
1. **Install OpenSSL**

   * OpenSSL is a free tool used to test SSL/TLS connections.
   * Download from: [https://www.openssl.org](https://www.openssl.org)

2. **Open Your Terminal**

   * Navigate to the folder where OpenSSL is installed (if needed).

3. **Connect to the Target Website via SSL**

   * Run this command:

     ```
     openssl s_client -host <target website> -port 443
     ```
   * Replace `<target website>` with the domain you want to test (e.g., `certifiedhacker.com`).

4. **Send an HTTP Request**

   * After the SSL connection is established, type:

     ```
     GET / HTTP/1.0
     ```
   * Then press **Enter** twice.

5. **View the Server Banner**

   * The server will respond with some info (like `Server: Apache`), which shows what software it's using.

## Detecting Web App Firewalls and Proxies on Target Site 
While footprinting the web infrastructure, attackers must discover the web application firewall and proxy settings of the target site to know the security measures employed. 

### Detecting Proxies 
To detect if a target web server is behind a proxy, attackers can use the **TRACE** method in an HTTP/1.1 request. This method makes the server return the exact request it received. If a proxy is in place, it may modify the request by adding headers before forwarding it to the actual server. When the response comes back, the attacker compares it with the original request to see if any changes were made, revealing the presence of a proxy.

**Example command using telnet:**
```bash
telnet targetsite.com 80
```
Then send:
```
TRACE / HTTP/1.1
Host: targetsite.com
```
This helps observe if any extra headers (like `Via` or `Forwarded`) were added by a proxy in the response.

### Detecting Web Application Firewalls
A Web Application Firewall (WAF) protects websites by filtering and blocking harmful traffic between the user and the web server. Attackers try to detect if a WAF is in place by analyzing how it behaves. One way is by checking cookies—some WAFs add unique cookies during communication. Another way is by looking at HTTP headers, as WAFs often modify them. By sending specific requests and studying the server's response, attackers can identify whether a WAF is active, helping them plan further attacks or bypass techniques.

Attackers use various tools such as WAFW00F to detect the presence of a WAF in front of a web server that hosts the target website.

Some WAF detection tools are listed below:
- **WAFW00F** [https://github.com/EnableSecurity/wafw00f]
- **WhatWaf** [https://github.com/Ekultek/WhatWaf]
- **Nmap** [https://nmap.org]
- **Web Application Firewall Detector** [https://pentest-tools.com]
- **SHIELDFY Web Application Firewall Detector** [https://github.com/shieldfy/waf-detector]
- **Advanced Web application firewall detection** [https://www.nmmapper.com]

## Hidden Content Discovery 
Hidden Content Discovery is a technique where attackers look for parts of a web application that aren’t directly visible or linked, such as old backups, configuration files, logs, or unfinished features. If found, these hidden elements can expose sensitive data or give attackers access to functions they shouldn’t have, making it easier to exploit the application and its users.

The following methods are employed for discovering the hidden content: 

### Web Spidering/Crawling 
Web spidering, or crawling, is a technique where automated tools scan a website to gather detailed information like email addresses, employee names, and hidden pages. Attackers use this data for footprinting and social engineering. The spider starts with a given URL and explores all linked pages by sending multiple requests and analyzing the HTML content to find more links. This process continues recursively, helping attackers map out the entire structure of the website, including directories and files. Although websites can block such crawling using a **robots.txt** file, attackers often ignore it to discover potential vulnerabilities and attack surfaces.

Some additional web spidering/crawling tools are as follows: 
- **OWASP Zed Attack Proxy** [https://www.zaproxy.org] - OWASP Zed Attack Proxy (ZAP) is an integrated penetration testing tool for finding vulnerabilities in web applications. It offers automated scanners as well as a set of tools that allow you to find security vulnerabilities manually. Attackers use OWASP ZAP for web spidering/crawling to identify hidden content and functionality in the target web application.

- **Web Data Extractor Pro** [http://www.webextractor.com]

### User-Directed Spidering
User-Directed Spidering is a manual way attackers explore a target website using a regular browser instead of automated tools. As they navigate through the site, they use tools like Burp Suite or WebScarab to monitor and capture all traffic between the browser and the server. These tools act like smart spiders and interceptors, building a detailed map of the site’s structure, links, and hidden features based on how the site responds. This method helps attackers discover areas of the application that may be missed by automated scanners.

## Detect Load Balancers
Organizations use load balancers to spread web traffic across multiple servers, improving performance and reliability. There are mainly two types—DNS-based (Layer 4) and HTTP-based (Layer 7) load balancers. Attackers can detect these using tools like `dig` or `lbd` (load balancing detector) to find out if a website resolves to multiple IP addresses, which reveals the presence of load balancing and may help them identify the actual IPs of backend servers.

### Using host command
Type the following host command to determine whether the target domain is resolving to multiple IP addresses:
```bash
host <target domain>
```

## Detecting Web App Technologies 
Detecting web app technologies is a common step attackers take during the reconnaissance phase to gather information about how a website is built. By identifying frameworks, programming languages, server types, and other components used, attackers can better understand the site’s structure and find possible weaknesses. This information helps them plan more targeted and effective attacks later on.

The following are various tools used by attackers to detect web application technologies: 

### Wappalyzer 
🔗Source: [https://www.wappalyzer.com] <br> 
Wappalyzer helps attackers discover the technology stack of any website, such as the content management system (CMS), e-commerce platform, or payment processor, along with company and contact details. The tool offers online search services and a browser extension to perform this task. 
- Open the Chrome Web Store [https://chromewebstore.google.com], search for **Wappalyzer**, and click **Add to Chrome**.
- Once added to the Chrome browser, it creates an icon in the top-right corner.

## WebSockets Enumeration 
While footprinting web infrastructure, attackers try to enumerate the WebSockets on the target site to identify the running WebSocket servers. Tools such as STEWS (Security Testing and Enumeration of WebSockets) help attackers accomplish this. 

### STEWS 
🔗Source: [https://github.com/PalindromeLabs/STEWS] <br> 
STEWS is a suite of tools for testing WebSockets, providing attackers with the ability to discover WebSocket endpoints, fingerprint the server running on an endpoint, and identify known vulnerabilities. To perform WebSocket enumeration on a target web application, run the following command:
```bash
python3 STEWS-fingerprint.py -1 -k -u <Target URL>
```
