# 🚫 Attack Authorization Schemes
Attack on authorization schemes happens when a web application’s access control is poorly implemented, allowing attackers to bypass restrictions after logging in. For example, if an admin page’s URL can be accessed without proper checks, an attacker could escalate privileges and view sensitive data. Such flaws let attackers access information or functions they are not authorized to use, even without valid credentials.

---
## Authorization Attack 
An authorization attack happens when an attacker logs in with a valid but low-privileged account and then exploits weaknesses to gain higher access. This is often done by altering data in HTTP requests—such as changing user IDs, access groups, file names, or parameters in URLs, POST data, cookies, and hidden fields. For example, by modifying a user ID in the query string, an attacker might view another user’s private data, or by tampering with a file name parameter, they could access restricted files.

1. **Uniform Resource Identifier (URI)** → A unique address used to identify resources. Attackers can use it to access restricted files, inject malicious queries, or redirect users to malicious sites.

2. **Parameter Tampering** → Changing parameters between client and server (in cookies, URLs, or hidden fields) to alter sensitive data like prices, permissions, or credentials.

3. **POST Data** → Manipulating data sent via POST requests, which often contains session or authorization details, to gain unauthorized access or control.

4. **HTTP Headers** → Altering HTTP request headers (using custom tools) to bypass security, inject malicious data, or exploit access controls.

5. **Query String and Cookies** → Modifying cookies or URL query strings to change session tokens, user preferences, or authorization details for malicious purposes.

6. **Hidden Tags** → Editing hidden form fields in HTML pages to manipulate application behavior and gain unauthorized access or control.

---
## Authorization Attack: HTTP Request Tampering
An HTTP request tampering attack happens when an attacker alters the data sent between a web client and server before it is processed. By modifying elements like HTTP headers, URLs, or form data, the attacker can manipulate how the server handles the request without needing another user’s ID, potentially bypassing authorization controls.

## Authorization Attack: Cookie Parameter Tampering 
Cookie parameter tampering is a method used to tamper with the cookies set by the web application to perform malicious attacks. When the user logs into the site, the web application sets the session cookie and stores it in the browser. 

Cookie parameter tampering is performed in the following steps: 

1. **Collect & Study Cookies** → First, gather session cookies from the website and check how they are made or generated.
2. **Modify & Replay Cookies** → Capture a session cookie, change its details using tools like Burp Suite, and send it back to the site to try accessing another user’s account.
3. **Intercept & Replace Cookies** → Use a tool to catch each browser request, swap the real cookie with your edited one, and test if weak cookie security lets you guess valid values.
