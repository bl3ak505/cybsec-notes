# 📱 Mobile-based Social Engineering Techniques 
**Mobile-based Social Engineering** is when attackers create fake versions of popular mobile apps with attractive features to trick users. These malicious apps are uploaded to app stores with names similar to real ones. When users download them thinking they’re genuine, the malware gets installed and infects their devices.

Listed below are some techniques attackers use to perform mobile-based social engineering:
- Publishing malicious apps
- Repackaging legitimate apps
- QRLJacking
- Using fake security applications
- SMiShing (SMS Phishing)

---
## Publishing Malicious Apps
In mobile-based social engineering, the attacker performs a social engineering attack using malicious mobile apps. The attacker first creates the malicious application — such as a gaming app with attractive features — and publishes it on major application stores using the popular names. Unaware of the malicious application, a user will download it onto their mobile device, believing it to be genuine. Once the application is installed, the device is infected by malware that sends the user’s credentials (usernames, passwords), contact details, and other information to the attacker.

## Repackaging Legitimate Apps 
**Repackaging Legitimate Apps** is a trick where attackers take a real app—like a popular game—add hidden malware to it, and upload it to third-party app stores. When users download the modified app, the malware silently collects personal data from their devices and sends it to the attacker. The app looks normal, but it’s actually spying on the user in the background.

## Fake Security Applications
**Fake Security Applications** are used by attackers as part of mobile-based social engineering. First, they infect the victim’s computer and then upload a fake app to an app store. When the victim tries to log in to their bank account, a fake pop-up tells them to download a “security app” on their phone. Believing it’s legit, the victim installs the app, which secretly steals their banking credentials. When the bank sends a second-factor authentication code via SMS, the attacker uses it to access and take over the victim’s bank account.

## SMiShing (SMS Phishing) 
**SMiShing (SMS Phishing)** is a mobile-based scam where attackers send fake text messages to trick users into taking quick actions—like clicking a malicious link, calling a fake number, or sharing personal information. These messages often appear urgent or important, pretending to be from trusted sources like banks or companies.

For example, **Tracy**, a software engineer, receives an SMS claiming to be from **XIM Bank’s security team**, asking her to call a number urgently. Believing it's real, she calls and is asked to share her card number and password. Thinking it's genuine, she provides the details, unknowingly giving them to a scammer.

SMiShing tricks can also include fake prize messages asking users to pay a small fee and share personal info.

## QRLJacking
**QRLJacking** is a social engineering attack that targets QR code login systems on websites. Attackers trick users into scanning a fake QR code, which looks like a real login code. They first get a real QR code from a website, clone it, and place it on a phishing page that updates automatically.

When the victim scans the fake QR code with their mobile app, their login details and device info (like GPS, IMEI, SIM data) are sent to the attacker. This gives the attacker full access to the victim’s account and allows them to perform harmful actions like stealing data, impersonation, or making illegal transactions.

### QR Code Cloning Tools
Cloning the QR code captured from the legitimate website is a crucial step in a QRLJacking attack. Attackers can use tools such as QR TIGER to clone a legitimate QR code for malicious purposes. 

### QR TIGER 
🔗Source: [https://www.qrcode-tiger.com]

QR TIGER is a QR code generator tool that allows attackers to create a duplicate copy of a legitimate QR code. It can clone static as well as dynamic QR codes and generate multiple duplicates of the QR code using a folder. Furthermore, the tool allows attackers to track information on dynamic QR codes such as the number of scans, the time and location of scans, and the device used to scan from different locations, which can help in monitoring the target user.
