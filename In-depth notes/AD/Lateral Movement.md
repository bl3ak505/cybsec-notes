# 📌 Lateral Movement

> Post-exploitation phase where you pivot from your initial foothold across the network — usually using legitimate protocols with stolen credentials so it looks like normal admin traffic.

---

## 🔍 Definition

Lateral movement is the set of post-exploitation techniques used to expand access across a network after initial compromise. Attackers move from one host to another, accumulate more credentials, and work toward high-value targets (Domain Controllers, file servers, backups). The core challenge: these techniques abuse **legitimate protocols** (SMB, RDP, WinRM, WMI) with valid credentials, making them nearly indistinguishable from normal administrative traffic. MITRE ATT&CK: **TA0008**.

In AD environments, lateral movement is the phase between "I have a low-priv shell on one workstation" and "I own the domain."

---

## 🧠 Core Concepts

- **Credential Reuse** — same password across multiple machines. Very common. One cred = many pivots
- **Pass-the-Hash (PtH)** — reuse stolen NTLM hash to authenticate without cracking (see [[NTLM and Pass-the-Hash]])
- **Pass-the-Ticket (PtT)** — reuse stolen Kerberos TGT/TGS tickets (see [[Kerberos Authentication]])
- **Living off the Land (LOTL)** — use built-in Windows tools (PsExec, WMI, PowerShell) to avoid dropping custom malware. According to Picus Security 2025, PowerShell appeared in **71% of LOTL attacks**
- **Token Impersonation** — steal/impersonate access tokens of other users running on the same machine (Potato attacks, `SeImpersonatePrivilege`)
- **Admin Shares** — hidden Windows shares (`ADMIN$`, `C$`, `IPC$`) used by PsExec and other tools to copy binaries and execute remotely
- **DCOM** — Distributed COM. Another remote execution channel, stealthier than SMB
- **SMB Signing** — when disabled, allows NTLM relay attacks (attacker relays captured NTLM auth to other hosts)

## ⚙️ Lateral Movement Methods

### 1. PsExec / SMBExec (Port 445)
Copies a service binary to `ADMIN$`, creates + starts a service, executes commands, removes it.
- **PsExec** — noisier. Creates `PSEXESVC` service. Event IDs: 7045 (service install), 4624 (logon type 3)
- **smbexec** — no binary drop. Uses `cmd.exe` via BAT file. Slightly stealthier

### 2. WMI / WMIExec (Port 135 + dynamic)
Uses Windows Management Instrumentation to spawn processes remotely. Fileless, runs under `WmiPrvSE.exe`. No service creation.

### 3. WinRM / PowerShell Remoting (Port 5985/5986)
Windows Remote Management — PowerShell-native. `Invoke-Command`, `Enter-PSSession`. WinRM must be enabled on the target.

### 4. RDP (Port 3389)
Full graphical session. Extremely common in ransomware deployments. Detectable (logon type 10).

### 5. DCOM
Abuse of COM object interfaces (`MMC20.Application`, `ShellWindows`) for remote code execution. Stealthy — uses DCOM/RPC.

### 6. Pass-the-Hash
Authenticate via NTLM hash without knowing the plaintext. Works with PsExec, WMI, WinRM, CME, evil-winrm.

### 7. Pass-the-Ticket
Steal and inject Kerberos tickets to access services as another user.

### 8. SSH (Linux/Unix environments)
SSH keys or password reuse across Linux servers. SSH session hijacking via `ControlMaster`.

## 📖 Key Terminology

| Term | Meaning |
|------|---------|
| ADMIN$ | Hidden share pointing to `%SystemRoot%`. Required for PsExec |
| C$ | Hidden share for the C drive root |
| IPC$ | Inter-Process Communication share. Used for named pipes and DCOM |
| Logon Type 3 | Network logon. Generated by PsExec, SMB, WMI |
| Logon Type 10 | RemoteInteractive (RDP) |
| WmiPrvSE.exe | WMI provider host — parent process of WMI-spawned commands |
| LocalAccountTokenFilterPolicy | Registry key. Set to 1 = enables PtH for non-RID500 local admins |
| Cobalt Strike | C2 framework with built-in lateral movement (SMB beacon, PsExec, WMI) |
| Impacket | Python suite: wmiexec, smbexec, psexec, dcomexec — all support hash auth |

## 🔥 Important Details & Gotchas

- **UAC token filtering breaks PtH for non-RID500 local accounts** — if the target machine has UAC enabled and you're using a non-default local admin account, PtH via SMB will fail (returns "Access Denied" on `ADMIN$`). Only the built-in Administrator (RID 500) bypasses this. Fix: `LocalAccountTokenFilterPolicy = 1`
- **Domain admin via PtH always works** — UAC filtering doesn't apply to domain accounts that are in the local admins group
- **WMI is stealthier than PsExec** — no service creation (Event ID 7045). But still generates Event ID 4624 + process creation events
- **smbexec vs wmiexec** — smbexec is noisier (service creation), wmiexec is cleaner but needs RPC/DCOM open. For stealth, prefer wmiexec or WinRM
- **TGT tickets expire in 10 hours** — pass-the-ticket has a window. NTLM hashes last until password change
- **PowerShell in 71% of LOTL attacks** — but Script Block Logging (Event ID 4104) will catch it unless you bypass AMSI
- **NTLM relay** — if SMB signing is disabled, you can relay captured NTLM auth to other hosts. Responder + ntlmrelayx is the standard combo

## 🌍 Real-World Usage / Examples

**PsExec (Impacket):**
```bash
impacket-psexec corp.local/Administrator:pass@<TARGET-IP>
# With hash:
impacket-psexec corp.local/Administrator@<TARGET-IP> -hashes :<ntlmhash>
```

**WMIExec (stealthier):**
```bash
impacket-wmiexec corp.local/Administrator@<TARGET-IP> -hashes :<ntlmhash>
```

**SMBExec:**
```bash
impacket-smbexec corp.local/Administrator@<TARGET-IP> -hashes :<ntlmhash>
```

**Evil-WinRM (WinRM lateral movement):**
```bash
evil-winrm -i <TARGET-IP> -u Administrator -H <ntlmhash>
evil-winrm -i <TARGET-IP> -u Administrator -p 'pass'
```

**CrackMapExec (spray + execute):**
```bash
# Spray hash across subnet
crackmapexec smb 10.10.10.0/24 -u Administrator -H <ntlmhash> --local-auth

# Execute command
crackmapexec smb <TARGET-IP> -u Administrator -H <ntlmhash> -x 'whoami'

# Check WinRM access
crackmapexec winrm <TARGET-IP> -u Administrator -H <ntlmhash>
```

**Pass-the-Ticket:**
```bash
# Dump tickets from LSASS (Mimikatz, Windows)
sekurlsa::tickets /export

# Inject ticket (Kali)
export KRB5CCNAME=ticket.ccache
impacket-psexec -k -no-pass corp.local/Administrator@<DC>
```

**NTLM Relay (no creds → lateral movement):**
```bash
# Terminal 1: Start relay
impacket-ntlmrelayx -tf targets.txt -smb2support

# Terminal 2: Responder (capture and relay)
sudo responder -I eth0 -rdwv
# When user authenticates to a fake share, their auth is relayed to targets.txt
```

**Real-world context:** CONTI ransomware used Mimikatz + PsExec to spread across thousands of hosts in minutes. Fog ransomware (2024-2025) relied on PowerShell + WMI for LOTL lateral movement in educational institutions.
